# Автоматизация тестирования проекта Blago (`blago_test`)

Проект содержит UI и API автотесты для сайта blago.ru. Фреймворк построен на базе Python, pytest и Selenium с использованием паттерна Page Object Model.

---

## Структура проекта

* **`api/`** — модули для работы с API.
    * `api_client.py` — базовый клиент для работы с HTTP-запросами.
    * `response_utils.py` — вспомогательные утилиты для валидации ответов сервера.
* **`pages/`** — реализация паттерна Page Object Model (для UI-тестов).
    * `base_page.py` — базовый класс с общими методами для всех страниц.
    * `main_page.py`, `action_page.py`, `donation_form_page.py` — классы конкретных страниц сайта с их локаторами и логикой взаимодействия.
* **`tests/`** — папка со всеми тест-кейсами, разделенная по уровням тестирования:
    * `api/` — тесты для проверки работоспособности API (эндпоинты, статус-коды).
    * `ui/` — тесты пользовательского интерфейса (навигация, формы пожертвований, валидация полей).
    * `integration/` — интеграционные сценарии (проверка связи UI-действий с изменениями в API и наоборот).
* **`utils/`** — вспомогательные инструменты(генерация тестовых данных, работа с конфигурационными файлами).
    * `config.py` — конфигурация окружения и настроек.
* **`conftest.py`** — конфигурационный файл pytest. Здесь настраиваются фикстуры (запуск/закрытие браузера, инициализация клиентов).
* **`pytest.ini`** — основной файл настроек тест-раннера pytest.
* **`requirements.txt`** — список всех внешних зависимостей проекта (библиотек), необходимых для работы тестов.
* **`.gitignore`** — файл, указывающий Git, какие файлы и папки не нужно отслеживать (кэш, отчеты, локальные настройки).

## Стек технологий

* **Язык:** Python 3.14+
* **Тест-раннер:** pytest
* **UI Автоматизация:** Selenium WebDriver
* **Отчетность:** Allure Framework

---

## Быстрый запуск

### 1. Клонирование репозитория и переход в папку
```bash
git clone [https://github.com/MaksTrofimov/blago_test.git]
cd blago_test
```

### 2. Настройка виртуального окружения
```bash
# Создание окружения
python -m venv .venv

# Активация для macOS/Linux (твой случай)
source .venv/bin/activate

# Активация для Windows (на всякий случай)
.venv\Scripts\activate
```

### 3. Установка зависимостей
```bash
pip install -r requirements.txt
```
### 4. Запуск тестов
* Запустить все доступные тесты
  ```bash
  pytest
  ```
* Запустить только тесты для API
  ```bash
  pytest tests/api/
  ```
  * Запустить только тесты для UI
  ```bash
  pytest tests/ui/
  ```
  * Запустить только интеграционные сценарии
  ```bash
  pytest tests/integtarion/
  ```

  ### 5. Генерация отчетов Allure
  ```bash
  allure serve allure-results
  ```
